# 帳務管理系統 v2 — 開發紀錄與接續指引

## 專案概述
為豐裕國際倉儲股份有限公司開發的桌面帳務管理系統。
使用 Electron + SQLite 打包成免安裝版，支援 Windows 和 Mac。

---

## 技術架構
- **框架**：Electron v28
- **資料庫**：SQLite（better-sqlite3）
- **AI 模型**：`claude-sonnet-4-6`（每次更新都要確認這個名稱正確）
- **第三方整合**：Google Drive API、Google Sheets API（OAuth 2.0）
- **打包方式**：免安裝版

## 功能清單

### AI 工具
- **拍照辨識**：上傳圖片或開啟鏡頭直接拍照，AI 辨識出帳單據
  - 預設類型：出帳
  - AI 提示詞：直接從圖片文字讀取廠商名稱，不推測
  - 廠商辨識錯誤可在出帳管理點「編輯」修改
- **Excel 匯入**：
  - 下載固定格式範本填寫後上傳（不需 API Key）
  - AI 智慧匯入任意格式（需 API Key）

### 帳務
- **客戶付款追蹤**（最上方）：記錄應收帳款，確認付款後自動同步至入帳管理，來源標示「付款確認」
- **入帳管理**：手動新增或從客戶付款追蹤同步
- **出帳管理**：手動新增或 AI 辨識，每筆都有「編輯」功能可修改錯誤
- **所有紀錄**：整合入帳出帳，可依月份篩選，支援匯出 CSV

### 設定
- **Google Drive 同步**：自動掃描「豐裕請款單」資料夾抓請款單
- **客戶名單**：分「入帳客戶」和「出帳廠商」兩類

## Google Drive 抓取規則（重要）

- **目標資料夾**：「豐裕請款單」
- **跳過**：「已結案」資料夾、「請款單總表」、「倉儲收入報表」、「廠商資料」、「歷史紀錄」
- **月份判斷**：
  - 優先從工作表名稱判斷（例：`115 04`、`11504`、`新版清清11504`）
  - 其次從計費期間欄位（例：`115/04/01-04/30`）
  - 請款日期不用來判斷月份（下個月才請款是正常的）
- **金額**：只抓「應付總金額」，沒有才用「總金額(含稅)」
- **客戶名稱**：從試算表內容的「客戶名稱：」欄位讀取
- 年份選單：民國 112～130 年

---

## 客戶名單規則

- 分「入帳客戶」和「出帳廠商」**完全分開**，不合併
- 自動新增時機：
  - 手動/AI 新增入帳 → 自動加入「入帳客戶」
  - 手動/AI 新增出帳 → 自動加入「出帳廠商」
  - Google Drive 同步時 → 自動新增新客戶到入帳客戶
  - 重複的不再新增

## Anthropic API

- **模型名稱**：`claude-sonnet-4-6`（重要！更新程式時確認 main.js 裡的模型名稱）
- **API Key 設定**：程式「拍照辨識」頁面 → 設定 API Key
- **費用估算**：
  - 拍照辨識一張：約 NT$0.1～0.2
  - Google Drive 請款單分析一張：約 NT$0.02
  - 一個月大概 NT$5～20

## 每月請款單匯入流程

**方式一（推薦）：Google Drive 自動同步**
1. 程式「Google Drive 同步」→ 選年份月份 → 開始抓取
2. 預覽確認後匯入到「客戶付款追蹤」

**方式二：手動上傳給 Claude 整理**
1. 從 Google Drive 下載請款單資料夾打包成 zip
2. 上傳給 Claude，說明要哪個年份月份
3. Claude 整理出 CSV（客戶名稱、月份、應付總金額）
4. 在程式「客戶付款追蹤」→「選擇 CSV 匯入」

## 檔案結構

```
accounting-app-v2/
  main.js          ← 主進程：IPC handlers、DB 操作、路徑設定
  preload.js       ← 安全橋接，暴露 API 給前端
  google-auth.js   ← Google OAuth 授權流程（本地 3141 port 接收 code）
  google-drive.js  ← Google Drive 抓取、Excel 解析邏輯
  package.json     ← 依賴套件、打包設定（含 Mac 鏡頭權限聲明）
  src/
    index.html     ← 介面 HTML
    renderer.js    ← 前端 JS 邏輯
```
